# Экран «Сегодня» - подробный макет (этап D.1)

**Назначение:** полная спецификация главного экрана приложения. Используется для отрисовки в Figma и затем один-в-один переносится во Flutter.  
**Контекст:** это экран, с которого начинается 80 % сессий. Всё, что мешает **за 2 секунды** понять «что принять и когда» - лишнее.  
**Источники:** `UI_DESIGN_PLAN.md` §1 (экран №3), `DESIGN_GUIDELINES_PARKINSON.md` §1-7, токены из `UI_DESIGN_PLAN.md` §B.

---

## 0. Назначение экрана

**Ответ на один вопрос пациента:** *«Что мне принять прямо сейчас или ближайшее?»*

Всё остальное (история, список лекарств, настройки) - через нижнюю навигацию, не на этом экране.

**Ключевые метрики успеха макета:**

- Главная кнопка действия («Принял сейчас») достижима **в 1 тап** с момента открытия экрана.
- Информация о ближайшем приёме читается **без прокрутки** на экране 360×640 dp (минимальный baseline).
- Статус любого приёма понятен **без цвета** (проверка ч/б).

## 2. AppBar - детально

```
┌──────────────────────────────────────────┐
│                                          │
│  Сб, 25 апреля               [🖐]  [🔔]  │
│  text.title.m (22 sp, 600)               │
│                                          │
└──────────────────────────────────────────┘
```

| Элемент | Размер | Токен | Поведение |
|---------|--------|-------|-----------|
| Дата | 22 sp, weight 600, `color.text.primary` | `text.title.m` | статическая строка вида «Сб, 25 апреля» |
| Иконка `palm` (режим «Большая кнопка») | 48×48 dp, область нажатия 64×64 dp | - | открывает экран №5 |
| Иконка `bell` (напоминания / настройки звука) | 48×48 dp, область нажатия 64×64 dp | - | открывает раздел настроек звука |
| Высота AppBar | 56 dp | - | без теней |
| Нижняя граница | линия 1 dp, `color.border.default` | - | заменяет тень (§3.3 гайдлайна: избегаем размытия) |

**Почему иконки без подписей** (нарушает §2.3 гайдлайна?) - в AppBar их две, они **системно-служебные**, не «входные точки в фичи»; основные точки входа - в BottomNav, где подписи обязательны. Исключение зафиксировать в `design/a11y.md`.

## 3. Карточка «Следующий приём» - базовая структура

Это **главный элемент экрана**. Один и тот же компонент в 4 состояниях.

```
┌──────────────────────────────────────────┐  ← space.l (24 dp) сверху
│ ┌──────────────────────────────────────┐ │
│ │                                      │ │  ← card padding 24 dp
│ │         Следующий приём              │ │  ← text.title.m (22 sp)
│ │                                      │ │     color.text.secondary
│ │                                      │ │  ← space.m (16 dp)
│ │          14 : 32                     │ │  ← text.display (96 sp, 600)
│ │                                      │ │     tabular-nums
│ │       через 1 ч 12 мин               │ │  ← text.title.m (22 sp)
│ │                                      │ │     color.text.secondary
│ │                                      │ │  ← space.l (24 dp)
│ │     Леводопа   250 мг                │ │  ← text.title.l (28 sp, 600)
│ │     1 таблетка, после еды            │ │  ← text.body (20 sp)
│ │                                      │ │  ← space.l (24 dp)
│ │  ┌────────────────────────────────┐  │ │
│ │  │   ✓  Принял сейчас             │  │ │  ← Button / Primary, 88 dp
│ │  └────────────────────────────────┘  │ │
│ │                                      │ │  ← space.m (16 dp)
│ │  ┌──────────────┐  ┌──────────────┐  │ │
│ │  │  ⏱ +15 мин   │  │  ✕ Пропуск   │  │ │  ← Button / Secondary, 72 dp
│ │  └──────────────┘  └──────────────┘  │ │
│ │                                      │ │
│ └──────────────────────────────────────┘ │
└──────────────────────────────────────────┘  ← space.l (24 dp) снизу
```

**Размеры:**

- Ширина карточки = ширина экрана − 2 × `space.m` (16 dp слева и справа).
- Padding внутри карточки: 24 dp со всех сторон.
- Радиус: `radius.card` = 20 dp.
- Рамка: 1.5 dp `color.border.default` (меняется по состоянию - см. §4).

**Выравнивание:**

- Все тексты внутри карточки - **по центру** (чтобы центр внимания был один, а не «левый край + правый край»).
- Кнопки - первая на всю ширину, вторичные - равные друг другу, с `space.s` (8 dp) между ними.

**Тактильный отклик кнопок:**

- «Принял сейчас» → лёгкий haptic + галочка растёт из центра 180 мс + тост «Принято в 14:33» → карточка уезжает вниз (анимация 200 мс).
- «+15 мин» → короткий haptic + время в карточке пересчитывается без сброса экрана.
- «Пропуск» → **диалог подтверждения** (§5.3 гайдлайна).

## 4. Четыре состояния карточки

### 4.1. Состояние «Ждёт» (`waiting`)

**Условие:** до ближайшего приёма > 10 минут.

```
┌──────────────────────────────────────────┐
│ ┌──────────────────────────────────────┐ │   рамка 1.5 dp
│ │         Следующий приём              │ │   color.border.default
│ │                                      │ │
│ │          14 : 32                     │ │   text.display
│ │       через 1 ч 12 мин               │ │   color.text.secondary
│ │                                      │ │
│ │     Леводопа   250 мг                │ │   color.text.primary
│ │     1 таблетка, после еды            │ │
│ │                                      │ │
│ │  ┌────────────────────────────────┐  │ │   Button / Primary
│ │  │   ✓  Принял сейчас             │  │ │   фон - color.accent.positive
│ │  └────────────────────────────────┘  │ │   (не приглушённый,
│ │                                      │ │    чтобы не учить пациента
│ │  ┌──────────────┐  ┌──────────────┐  │ │    «ждать правильного момента»)
│ │  │  ⏱ +15 мин   │  │  ✕ Пропуск   │  │ │
│ │  └──────────────┘  └──────────────┘  │ │
│ └──────────────────────────────────────┘ │
└──────────────────────────────────────────┘
```

**Особенности:**

- Фон карточки: `color.bg.card` (нейтральный).
- Рамка: 1.5 dp `color.border.default`.
- Никакой пульсации, никакого цветового акцента.
- Кнопка «Принял сейчас» **активна** - пациент может принять раньше, если так сказал врач. Это часто в ПД при режиме «by feel».
- Счётчик «через 1 ч 12 мин» обновляется раз в минуту.

---

### 4.2. Состояние «Пора» (`due`)

**Условие:** от −10 мин до +15 мин от времени приёма.

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓   рамка 3 dp
┃ ┌──────────────────────────────────────┐ ┃   color.accent.positive
┃ │         🔔  Пора принять             │ ┃   ← заголовок сменился
┃ │                                      │ ┃     + иконка bell
┃ │          14 : 30                     │ ┃
┃ │          сейчас                      │ ┃   ← текст сменился
┃ │                                      │ ┃     color.accent.positive
┃ │     Леводопа   250 мг                │ ┃     text.title.m, weight 600
┃ │     1 таблетка, после еды            │ ┃
┃ │                                      │ ┃
┃ │  ┌────────────────────────────────┐  │ ┃
┃ │  │   ✓  Принял сейчас             │  │ ┃
┃ │  └────────────────────────────────┘  │ ┃
┃ │                                      │ ┃
┃ │  ┌──────────────┐  ┌──────────────┐  │ ┃
┃ │  │  ⏱ +15 мин   │  │  ✕ Пропуск   │  │ ┃
┃ │  └──────────────┘  └──────────────┘  │ ┃
┃ └──────────────────────────────────────┘ ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
```

**Изменения относительно `waiting`:**

| Элемент | Было (waiting) | Стало (due) |
|---------|---------------|-------------|
| Рамка карточки | 1.5 dp `border.default` | **3 dp `accent.positive`** |
| Заголовок | «Следующий приём» | **«🔔 Пора принять»** |
| Подпись времени | «через 1 ч 12 мин» | **«сейчас»**, цвет `accent.positive`, weight 600 |
| Фон карточки | `bg.card` | `bg.card` (без тонирования, чтобы не снижать контраст текста) |
| Пульсация рамки | нет | **один раз при появлении**, 400 мс, scale 1.0 → 1.015 → 1.0 |

**Про пульсацию (§4.1 гайдлайна):**

- Пульсирует **один раз** при переходе из `waiting` в `due`.
- Дальше рамка стоит статично.
- При системном `disableAnimations` или настройке «Уменьшить анимацию» - без пульсации, только смена рамки.
- Никаких «мигающих» или «бегущих» акцентов.

**Звук/тактилька:** НЕ на этом экране. Звук принадлежит уведомлению (экран №4), не приложению.

### 4.3. Состояние «Опаздывает» (`overdue`)

**Условие:** > 15 минут после времени приёма, действие не выполнено.

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓   рамка 3 dp
┃ ┌──────────────────────────────────────┐ ┃   color.accent.negative
┃ │         ⚠  Опоздание                 │ ┃   ← заголовок + иконка
┃ │                                      │ ┃     color.accent.negative
┃ │          14 : 30                     │ ┃
┃ │     опоздание 18 минут               │ ┃   ← text.title.m, 600
┃ │                                      │ ┃     color.accent.negative
┃ │     Леводопа   250 мг                │ ┃
┃ │     1 таблетка, после еды            │ ┃
┃ │                                      │ ┃
┃ │  ┌────────────────────────────────┐  │ ┃
┃ │  │   ✓  Всё равно принять         │  │ ┃   ← текст сменился
┃ │  └────────────────────────────────┘  │ ┃
┃ │                                      │ ┃
┃ │  ┌──────────────────────────────┐    │ ┃   ← одна вторичная кнопка
┃ │  │  ✕ Отметить как пропущено    │    │ ┃     на всю ширину,
┃ │  └──────────────────────────────┘    │ ┃     72 dp
┃ │                                      │ ┃
┃ │  При опоздании посоветуйтесь        │ ┃   ← подсказка
┃ │  с врачом - не удваивайте дозу.     │ ┃     text.caption (18 sp)
┃ │                                      │ ┃     color.text.secondary
┃ └──────────────────────────────────────┘ ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
```

**Изменения относительно `due`:**

| Элемент | Стало (overdue) |
|---------|-----------------|
| Рамка | 3 dp `accent.negative` |
| Заголовок | «⚠ Опоздание» |
| Подпись времени | **«опоздание 18 минут»**, `accent.negative`, weight 600 |
| Основная кнопка | текст **«Всё равно принять»** (не «Принял сейчас» - честнее) |
| Вторичные кнопки | **одна** кнопка «Отметить как пропущено» во всю ширину; «+15 мин» убирается - отложить ещё на 15 мин уже бессмысленно |
| Подсказка под кнопками | «При опоздании посоветуйтесь с врачом - не удваивайте дозу.» |

**Почему убрали «+15 мин»:** в ПД двойная доза опасна, а «+15 мин» в состоянии опоздания провоцирует именно это поведение («ладно, потом»). Пациент должен сделать явный выбор: принять сейчас или честно отметить пропуск.

**Почему подсказка про врача:** это медицинское приложение. Снимает ответственность с продукта за непрямые последствия и одновременно - реально полезный текст.

### 4.4. Состояние «Принято» (`taken`) - переходное

**Условие:** после нажатия «Принял сейчас» или «Всё равно принять».

**Это не статичное состояние**, а короткий переход 1.5 секунды:

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓   рамка 3 dp accent.positive
┃                                          ┃
┃                                          ┃
┃             ┌─────────┐                  ┃
┃             │   ✓     │                  ┃   ← круг 96 dp
┃             └─────────┘                  ┃     accent.positive fill
┃                                          ┃     иконка check 48 dp
┃                                          ┃     animation: scale 0 → 1
┃          Принято в 14:33                 ┃     180 мс, ease-out
┃                                          ┃
┃          Леводопа   250 мг               ┃   ← text.title.m
┃                                          ┃     color.text.secondary
┃                                          ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
```

**Через 1.5 с:**

1. Карточка уезжает вниз (transform translateY, 200 мс, ease-in).
2. На её место поднимается следующий приём в состоянии `waiting` или `due`.
3. Принятый приём появляется первой строкой в секции «Уже приняты» с галочкой.

**При системном «Уменьшить анимацию»:** переход укорачивается до 80 мс cross-fade, без движения.

## 5. Секция «Сегодня ещё»

Список будущих приёмов этого дня (не включая карточку-фокус).

```
                                              ← space.l (24 dp)
Сегодня ещё                                   ← text.title.m (22 sp, 600)
                                              ← space.m (16 dp)
┌──────────────────────────────────────────┐
│ ○  18:00    Леводопа 250 мг              │  ← ListItem / Intake
├──────────────────────────────────────────┤     72 dp высотой
│ ○  22:00    Прамипексол 1 мг             │
└──────────────────────────────────────────┘
```

**Детали строки:**

| Элемент | Размер | Токен | Поведение |
|---------|--------|-------|-----------|
| Иконка статуса (`○`, `✓`, `✕`) | 32×32 dp | - | `○` - запланировано; `✓` - принято; `✕` - пропущено |
| Время | 22 sp, tabular-nums | `text.title.m` | `color.text.primary` |
| Название + дозировка | 20 sp | `text.body` | `color.text.primary` |
| Разделитель | 1 dp, `color.border.default` | - | между строками |
| Высота строки | **72 dp** | `size.touch.min` +8 | ≥ min touch target |
| Тап по строке | - | - | разворачивает быстрые действия «Принять / Отложить / Пропустить» в том же месте (accordion) |

**Свёрнутый accordion:**

```
│ ○  18:00    Леводопа 250 мг              │
```

**Развёрнутый:**

```
│ ○  18:00    Леводопа 250 мг              │
│             1 таблетка, после еды         │
│  ┌────────────────────────────────────┐  │
│  │   ✓  Принять сейчас                │  │   ← Button / Secondary 72 dp
│  └────────────────────────────────────┘  │
│  ┌──────────────┐  ┌──────────────┐      │
│  │  ⏱ +15 мин   │  │  ✕ Пропуск   │      │
│  └──────────────┘  └──────────────┘      │
```

**Почему accordion, а не отдельный экран:**

- В ПД каждый переход - стресс и дополнительный поиск кнопки «Назад».
- Контекст (время приёма, соседние строки) остаётся перед глазами.
- Один приём за раз - по-прежнему один фокус (§5.1 гайдлайна).

**Пустое состояние секции:**

```
Сегодня ещё
На сегодня больше напоминаний нет.
```

(20 sp, `color.text.secondary`, без карточки, без иконки).

## 6. Секция «Уже приняты»

```
                                              ← space.l (24 dp)
Уже приняты                                   ← text.title.m
                                              ← space.m
┌──────────────────────────────────────────┐
│ ✓  08:00    Леводопа 250 мг              │
├──────────────────────────────────────────┤

```

Та же вёрстка, что и «Сегодня ещё», только иконки другие и тап по строке **не раскрывает действия** (уже принято - отменить можно только из экрана «История»).

**Цвет иконки `✓`:** `color.accent.positive`, размер 32 dp.  
**Цвет иконки `✕` (если пропущено):** `color.accent.negative`, размер 32 dp, подпись «пропущено» рядом.

**Пустое состояние секции** (начало дня):

Секция не показывается вовсе, чтобы не забивать экран пустыми блоками.

## 8. Offline-баннер (поверх контента)

Когда `ConnectivityService` говорит «нет сети» - между AppBar и карточкой «Следующий приём» появляется полоса:

```
┌──────────────────────────────────────────┐
│ ⚠  Работаем из кэша. Данные сохраняются. │   56 dp
└──────────────────────────────────────────┘     Banner / Offline
                                                 bg: color.accent.warning
                                                 text: color.text.primary
                                                 20 sp
```

- Не блокирует интерфейс - всё работает.
- Иконка `⚠` + слово «кэша» - смысл без цвета.
- Убирается автоматически при возврате сети + короткий тост «Синхронизация завершена» на 2 секунды.

Соответствует §8 гайдлайна: «явный баннер без блокирующих диалогов».

## 9. Поведение скролла

- AppBar и карточка «Следующий приём» - **pinned** (всегда видны).
- Секции «Сегодня ещё» и «Уже приняты» - скроллятся под ними.
- Никаких эффектов parallax, stretch, bounce в заголовках (§4 гайдлайна).
- Скролл-бар виден только во время скролла, стандартный системный, не кастомный тонкий.

---

## 10. Адаптив к размеру шрифта

Базовый макет рассчитан на **20 sp** основного текста и **96 sp** таймера.

При выборе пользователем «Крупный» (×1.3 ≈ 26 sp body, 125 sp таймер):

- Таймер может **не помещаться** в одну строку на экране 360 dp - тогда уменьшаем до **76 sp** (не переносим на 2 строки, не обрезаем).
- Название лекарства (28 → 36 sp) - в 2 строки с `maxLines: 2`, ellipsis НЕ допускается. Если длинное название - показывается целиком с переносом.
- Подпись «1 таблетка, после еды» - `maxLines: 3`.
- Кнопка «Принял сейчас»: текст не влезает → высота увеличивается до 104 dp (никогда не ужимаем шрифт на primary-кнопке).

**При «Очень крупный» (×1.6):** карточка «Следующий приём» занимает весь экран, секции 3 и 4 уезжают под скролл - это ожидаемое поведение, не баг.

## 11. Поведение в тёмной теме

Правила отличий от светлой:

- Фон экрана: `#0E1013`, карточки: `#1A1D22`.
- Рамка карточки `waiting`: `#2B2F37` (1.5 dp) - слегка светлее фона, не «исчезающая».
- Акценты `positive` / `negative` - версии для тёмной темы (см. токены): `#4FD39A` и `#FF7A5C` - проверены на контраст с фоном карточки.
- Тёмная тема **никогда не «инверсия»** (автоматическая) - отдельно отрисованный макет.

## 12. Состояние «Нет лекарств» (empty screen)

Когда у пациента нет ни одного препарата в расписании:

```
┌──────────────────────────────────────────┐
│  Сб, 25 апреля              [🖐]  [🔔]   │
├──────────────────────────────────────────┤
│                                          │
│                                          │
│                🕐                         │   иконка 96×96 dp
│                                          │
│                                          │
│        Пока нет напоминаний               │   text.title.l (28 sp, 600)
│                                          │
│     Добавьте первое лекарство -          │   text.body (20 sp)
│     и я напомню точно вовремя.           │   color.text.secondary
│                                          │
│                                          │
│   ┌────────────────────────────────┐     │
│   │   +  Добавить лекарство        │     │   Button / Primary 88 dp
│   └────────────────────────────────┘     │
│                                          │
│                                          │
├──────────────────────────────────────────┤
│ [Сегодня] [Лекарства] [История] ...      │
└──────────────────────────────────────────┘
```

**Правила:**

- **Одна** кнопка, ведёт в мастер добавления препарата (экран №7).
- Никаких иллюстраций пожилых людей, таблеток, рук - это воспринимается как снисходительность.
- Иконка 🕐 - нейтральная, одна штука.
- AppBar и BottomNav не скрываются.

## 13. Таблица состояний и переходов

| Событие | Из состояния | В состояние | Откуда |
|---------|--------------|-------------|--------|
| Время подошло к T−10 мин | waiting | due | таймер |
| Пациент нажал «Принял сейчас» | waiting/due/overdue | taken → следующий приём | действие |
| Прошло > T+15 мин, действие не выполнено | due | overdue | таймер |
| Пациент нажал «+15 мин» | due/waiting | waiting (новое время T+15) | действие |
| Пациент нажал «Пропуск» → подтвердил | любое | next item, запись «пропущено» в историю | действие |
| Сеть пропала | любое | то же + offline banner | `ConnectivityService` |
| Нет лекарств | - | empty screen | репозиторий |

## 14. Чек-лист приёмки макета (§9 гайдлайна)

- [ ] Все кнопки ≥ 64 dp, primary ≥ 88 dp, расстояния ≥ 16 dp.
- [ ] Основной текст ≥ 20 sp (перекрывает минимум 18 sp).
- [ ] Таймер 96 sp, tabular-nums, читается на ходу.
- [ ] Каждый статус дублирован иконкой + словом: `✓ Принято`, `✕ Пропущено`, `○ Ждёт`, `🔔 Пора`, `⚠ Опоздание`.
- [ ] В ч/б режиме все 4 состояния карточки различимы по форме/иконке/тексту.
- [ ] Контраст текста на карточке в светлой теме ≥ 14:1 (AAA), в тёмной ≥ 12:1 (AAA).
- [ ] Пульсация при переходе в `due` - **один раз**, отключается системной настройкой.
- [ ] BottomNav со всеми 5 подписями.
- [ ] Офлайн - баннер, не диалог.
- [ ] Empty state - одна кнопка, никаких дополнительных экранов.
- [ ] При «Крупный» шрифт макет не ломается (без обрезания текста).
